## 4. Findings this session (the important ones)

1. **Strategy-field volatility (REAL BUG, fixed in `0206e7a`)**: the sale `S.strategy` field differs between SOL snapshots — golden BASE FILE says `KEMBA` where the live exports say `CAFEXPORT`/blank for the same contracts. Worse, the drifted value can collide with a different historical subgroup: 4 NESTRADE/AA sales would have been **silently mis-allocated** to blend #91 instead of #625. Fix in `src/lib/blends.ts`: any (client, grade) that ever took >1 blend is ALWAYS flagged regardless of strategy; loose (client, grade) fallback tier when the exact key misses. LOO: 35% auto / 65% flagged / **0% wrong**; demo flow: 40/61 auto, all matching Ivo's actual choices.
2. **Default-horizon bug (REAL BUG, fixed in `f11b631`)**: "drop oldest month, take 10 present months" broke on sparse months and a stray `2024/10` ship date (net came out −12,050 then +35,568 instead of −4,850). Now: 10 consecutive calendar months anchored at position month −6 (reproduces workbook E:N exactly). Override per call via `horizonMonths`.
3. **Excel SUMIFS is case-insensitive** — `<>Kenyacof` must also exclude `KENYACOF`; replicated in `futsspread.ts`.
4. **The golden workbook's futures pivot was STALE** (cached KCN/2026 −516.0 vs refreshed −544.77, KCK/2026 missing). We compute it live from sales — ours is the refreshed truth.
5. **SOL snapshot drift**: on-disk ReportLogistic ≠ workbook paste — SSKE-107893 SMT revised (−151.2 vs −144.9), SSKE-98454 only in the export, SSKE-103502 is a 2-row contract split (parser aggregates by contract+month).
